# v5 Rules

Lightweight, binding conventions for `packages/v5` and `examples/v5` — smaller
than an architectural decision (no alternatives or consequences to weigh),
but still worth writing down so they're followed consistently rather than
rediscovered ad hoc in review each time.

If a convention grows real alternatives and tradeoffs behind it, it belongs
in `docs/adr/` instead — see [ADR-0000](../adr/0000-madr-template-for-adr.md).

## Index

- [No unsafe unwrap in tests](./no-unsafe-unwrap-in-tests.md)

## Contributing

When a convention comes up in review that should apply consistently going
forward — not just to the one line it was raised on — write it down here:
one file per rule, a short "Rule" statement, why it matters, and a
before/after example. Add it to the index above.
